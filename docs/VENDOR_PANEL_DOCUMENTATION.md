# BOHOSAAZ Vendor Panel Documentation

This guide explains how a new vendor can join BOHOSAAZ, complete KYC, upload products, manage orders, handle returns, and use the vendor panel correctly.

## 1. Vendor Journey Overview

A vendor follows this journey from start to finish:

1. Create a customer account or log in.
2. Apply to become a vendor.
3. Fill shop details, pickup details, KYC details, bank details, and upload required documents.
4. Submit the application for admin review.
5. Wait for vendor and KYC approval.
6. After approval, open the Vendor Panel.
7. Create products and submit them for approval.
8. Manage orders, tracking, returns, settings, and support.

## 2. Before You Start

Keep these details ready before applying:

- Shop name and brand display name.
- Shop contact email and phone number.
- Shop address with city, state, and pincode.
- Pickup address, if different from the shop address.
- Brand logo.
- Store banner, optional.
- PAN number.
- GSTIN, optional if not applicable.
- Aadhaar last 4 digits, optional.
- Bank account name, account number, IFSC, and bank name.
- UPI ID, optional.
- PAN image.
- GST certificate, optional.
- Cancelled cheque or bank proof.
- Address proof.

Uploaded files should be clear and readable. Very large files may fail, so keep each file under the allowed upload size.

## 3. Create or Log In to Your Account

1. Go to the BOHOSAAZ website.
2. Create a new account if you are a new user.
3. Log in with your registered email/mobile and password.
4. Open your account area.
5. Select the option to apply as a vendor.

You must be logged in before submitting a vendor application.

## 4. Apply to Become a Vendor

Open:

`Account -> Vendor Apply`

The application form has four main sections:

- Shop information.
- Pickup address.
- KYC details.
- Bank and document details.

### 4.1 Shop Information

Fill the following details:

- `Shop Name`: Your official shop or brand name. This is required.
- `Display Name`: The public name shown to customers, if different from the shop name.
- `Contact Email`: Email used for vendor communication.
- `Contact Phone`: Phone number used for vendor communication.
- `Shop Description`: A short description of your brand, products, and specialty.
- `Brand Logo`: Upload a clear logo image. This is required.
- `Store Banner`: Upload a banner image for your shop page. This is optional.

Best practices:

- Use a clean logo with good resolution.
- Keep the shop description short, professional, and easy to understand.
- Avoid adding misleading claims or unsupported promises.

### 4.2 Shop Address

Fill the shop address:

- `Shop Address 1`: Main address line. This is required.
- `Shop Address 2`: Additional address details, optional.
- `City`: Required.
- `State`: Required.
- `Pincode`: Required.

This address helps the platform verify your shop location.

### 4.3 Pickup Address

Fill pickup details if products will be picked up from a different location:

- `Pickup Name`
- `Pickup Phone`
- `Pickup Address 1`
- `Pickup Address 2`
- `City`
- `State`
- `Pincode`

If your pickup location is the same as your shop address, you may leave optional pickup fields empty unless the platform asks for them.

Important:

- Keep pickup phone number active.
- Enter the exact pincode where courier pickup will happen.
- Wrong pickup details may delay shipping.

## 5. Complete KYC

KYC is required for vendor verification.

Open:

`Vendor -> KYC`

or complete it during:

`Account -> Vendor Apply`

### 5.1 Choose KYC Type

Select one:

- `Individual`: Use this if you are selling as an individual.
- `Business`: Use this if you are selling as a registered business.

### 5.2 Identity Details

For individual KYC:

- Enter `Full Name`.
- Enter `PAN Number`.
- Enter `Aadhaar last 4 digits`, optional.

For business KYC:

- Enter `Business Name`.
- Enter `PAN Number`.
- Enter `GSTIN`, optional if not applicable.

Make sure the name matches your documents and bank details where possible.

### 5.3 Bank Details

Fill the bank details:

- `Account Name`: Name on the bank account. Required.
- `Account Number`: Bank account number. Required.
- `IFSC`: Bank branch IFSC code. Required.
- `Bank Name`: Required.
- `UPI ID`: Optional.

Important:

- Check account number and IFSC carefully before submitting.
- Account name should match your KYC/business name as closely as possible.

### 5.4 Upload Documents

Upload these documents:

- `PAN Image`: Required.
- `GST Certificate`: Optional.
- `Cancelled Cheque / Bank Proof`: Required.
- `Address Proof`: Required.

Good document upload rules:

- Use clear images or PDFs.
- Make sure all text is readable.
- Do not crop important details.
- Upload the correct document in the correct field.

## 6. Submit Application

After filling all required fields:

1. Review all shop, KYC, bank, and document details.
2. Click `Submit Application`.
3. The application status becomes `PENDING`.
4. Admin will review the application and KYC.

After submission, you can check the current status from:

`Vendor -> KYC`

Possible vendor statuses:

- `NOT_APPLIED`: Application has not been submitted.
- `PENDING`: Application is submitted and waiting for admin review.
- `APPROVED`: Application is approved and vendor access is available.
- `REJECTED`: Application was rejected. You can review the reason and resubmit.
- `SUSPENDED`: Vendor access is blocked by admin.

Possible KYC statuses:

- `NOT_SUBMITTED`: KYC has not been submitted.
- `SUBMITTED`: KYC is submitted and waiting for review.
- `VERIFIED`: KYC is approved.
- `REJECTED`: KYC was rejected. Check the reason and resubmit.

## 7. If Application or KYC Is Rejected

If your application is rejected:

1. Open `Vendor -> KYC` or `Account -> Vendor Apply`.
2. Read the rejection reason.
3. Correct the wrong details or upload better documents.
4. Submit the application again.

Common rejection reasons:

- Blurry or unreadable documents.
- Wrong PAN number.
- Bank proof does not match bank details.
- Incomplete shop address.
- Invalid pickup address.
- Business name and document name mismatch.

## 8. Access the Vendor Panel

After approval, open:

`Vendor Panel`

Main sections available in the vendor panel:

- `Dashboard`
- `Orders`
- `Return / Refund`
- `All Products`
- `Create Product`
- `KYC`
- `Support`
- `Settings`

If approval is complete but access is not updated, use `Refresh Access` from the approved application screen, then open the vendor dashboard again.

## 9. Vendor Dashboard

The dashboard shows live vendor metrics. Use it to quickly understand your store performance.

Typical dashboard information includes:

- Sales metrics.
- Order activity.
- Product activity.

Use the dashboard daily to check whether any order, return, or product needs attention.

## 10. Upload a New Product

Open:

`Vendor -> Create Product`

or:

`Vendor -> All Products -> Create Product`

### 10.1 Basic Product Details

Fill:

- `Title`: Product name.
- `Slug`: Optional URL-friendly product name.
- `Short Description`: Optional short summary.
- `Description`: Full product details.
- `Category`: Select the correct product category.
- `Brand`: Select your brand, if available.

Best practices:

- Product title should be clear and searchable.
- Description should include material, size, use case, and care instructions.
- Select the correct category so customers can find the product.

### 10.2 Pricing and Stock

Fill:

- `Price`: Main selling price.
- `Sale Price`: Optional discounted price.
- `Stock`: Available quantity.
- `SKU`: Optional product code.
- `Barcode`: Optional.

Rules:

- Price must be greater than zero.
- Sale price should be lower than the main price if used.
- Stock should be accurate to avoid overselling.
- Use unique SKU codes for easier inventory management.

### 10.3 Product Images

You can upload product images or add an image URL.

Image tips:

- Use clear, high-quality images.
- Upload multiple angles if available.
- Use a plain background where possible.
- The first/primary image should show the product clearly.
- Avoid watermarks, phone numbers, or promotional text on images.

### 10.4 Product Attributes

Fill optional details where applicable:

- `Material`
- `Shipping Class`
- `Weight`
- `Dimensions`: Length, width, height.
- `Tags`: Comma-separated keywords.
- `Country of Origin`
- `Warranty`
- `Return Policy`
- `Size Options`
- `Color Options`

These details improve customer confidence and reduce return requests.

### 10.5 SEO Details

Optional SEO fields:

- `Meta Title`
- `Meta Keywords`
- `Meta Description`

Use simple keywords customers may search for, but avoid keyword stuffing.

### 10.6 Variants

Use variants when the same product has different sizes, colors, or stock.

Variant fields:

- `Size`
- `Color`
- `SKU`
- `Price`
- `Sale Price`
- `Stock`
- `Active`

Important rules:

- Each variant should have a unique SKU.
- Variant price must be valid.
- Variant stock must not be negative.
- Disable inactive variants instead of deleting them if they may return later.

### 10.7 Submit Product

After entering product details:

1. Review title, category, price, stock, images, and variants.
2. Save or create the product.
3. Product status may become `PENDING`.
4. Admin reviews the product.
5. Once approved, product status becomes `PUBLISHED` or `Approved`.

Product statuses:

- `DRAFT`: Product is saved but not published.
- `PENDING`: Product is waiting for admin approval.
- `PUBLISHED`: Product is approved and visible to customers.
- `REJECTED`: Product was rejected and needs correction.

## 11. Manage Existing Products

Open:

`Vendor -> All Products`

From this section you can:

- View all your products.
- Filter products by occasion, recipient, or availability.
- Check product status.
- Edit product details.
- Update product images.
- Set primary image.
- Delete products if allowed.
- Export product list as CSV.

When editing a product:

1. Open the product edit option.
2. Update only the required fields.
3. Save changes.
4. If changes require admin review, wait for approval.

Product management best practices:

- Keep stock updated.
- Remove or disable unavailable products.
- Update price only after checking margins.
- Keep images and descriptions accurate.
- Do not list duplicate products unnecessarily.

## 12. Manage Orders

Open:

`Vendor -> Orders`

This section shows orders that contain your products only.

For each order, you can view:

- Order ID.
- Order status.
- Date and time.
- Product name.
- Quantity.
- Price.
- Line total.
- Masked shipping details.
- Item status.
- Tracking courier.
- Tracking number.

### 12.1 Update Order Item Status

Use the status dropdown for each order item.

Common order item statuses may include:

- `PLACED`
- `PROCESSING`
- `PACKED`
- `SHIPPED`
- `DELIVERED`
- `CANCELLED`

Always update the status when work progresses so customers and admins can track the order correctly.

### 12.2 Add Tracking Details

If shipment is handled manually:

1. Enter courier name.
2. Enter tracking number.
3. Save or update the item.

If Delhivery AWB creation is available:

1. Click the Delhivery AWB option.
2. Confirm AWB creation.
3. The tracking number will be generated if courier setup is successful.

### 12.3 Export Order Data

Use export options to download:

- Vendor orders CSV.
- Vendor orders PDF.
- Order packing slip.
- Order item CSV, where available.

Use packing slips while preparing shipments.

## 13. Return and Refund Management

Open:

`Vendor -> Return / Refund`

This section shows return/refund requests for your products.

You can view:

- Return request ID.
- Order ID.
- Product title.
- Request amount.
- Current return status.
- Masked customer shipping information.
- Last updated time.

Open a return request to review details and take the required action.

Important:

- Customer contact details may be hidden for privacy.
- Review the return reason carefully.
- Follow platform policy before approving or rejecting a request.
- Manage pickup or return logistics as instructed by the admin/platform.

Return/refund exports may be available as CSV or PDF.

- Check KYC status.
- View submitted identity details.

Use support when you need help from admin.

- Product approval issues.
- KYC rejection questions.
- Order or shipment problems.
- Return/refund disputes.
- Technical issues in the vendor panel.

Use this checklist every working day:

- Check new orders.
- Update order item status.
- Pack pending orders.
- Add courier and tracking details.
- Check return/refund requests.
- Review low-stock products.
- Update unavailable products.
- Check pending product approvals.

## 18. Product Listing Quality Checklist

Before submitting any product, confirm:

- Product title is clear.
- Category is correct.
- Price and sale price are correct.
- Stock is accurate.
- Main image is clear.
- Description explains the product properly.
- Size/color variants are correct.
- SKU is unique.
- Return policy is filled if needed.
- Product does not violate platform policy.

## 19. Common Mistakes to Avoid

Avoid these mistakes:

- Uploading blurry KYC documents.
- Entering wrong bank account or IFSC.
- Using incomplete pickup address.
- Creating products without images.
- Adding wrong stock quantity.
- Using duplicate SKUs for variants.
- Forgetting to update order tracking.
- Ignoring return requests.
- Not checking rejection reasons before resubmitting.

## 20. Recommended Product Image Guidelines

For better conversion:

- Use bright and clear images.
- Show the full product.
- Add close-up images for fabric, texture, or design.
- Use the same image style for all products.
- Avoid misleading edits.
- Do not upload images that belong to another brand unless you have rights.

## 21. Simple Vendor Workflow Example

Example flow for a new vendor:

1. Vendor creates an account.
2. Vendor opens `Account -> Vendor Apply`.
3. Vendor fills shop, KYC, bank, and document details.
4. Vendor submits the application.
5. Admin approves vendor and KYC.
6. Vendor opens `Vendor -> Create Product`.
7. Vendor uploads product information and images.
8. Product goes for admin review.
9. Admin approves the product.
10. Customer places an order.
11. Vendor checks `Vendor -> Orders`.
12. Vendor packs the product and adds tracking.
13. Order is delivered.

## 22. FAQ

### Why can I not access the Vendor Panel?

Your vendor application may still be pending, rejected, or suspended. Check `Vendor -> KYC` or `Account -> Vendor Apply`.

### What should I do if KYC is rejected?

Read the rejection reason, correct the details, upload clear documents, and resubmit.

### Can I upload products before approval?

Vendor access is normally available after approval. Complete application and KYC first.

### Why is my product not visible on the website?

The product may be in `DRAFT`, `PENDING`, or `REJECTED` status. Only approved/published products are visible to customers.

### Can I change pickup details later?

Yes. Open `Vendor -> Settings`, update pickup details, and click `Save`.

### How do I contact admin?

Open `Vendor -> Support` and create a support ticket with all relevant details.

## 23. Final Notes for Vendors

Keep your vendor profile accurate, upload quality products, maintain correct stock, ship orders on time, and respond quickly to returns or support messages. A well-managed vendor panel helps improve customer trust, order completion, and long-term sales.
